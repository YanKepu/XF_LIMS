
## JSON标准
Json是tcp在客户端和服务端之间的桥梁，对于Json格式，lims系统应有个统一的标准，用于解析和传递。lims的服务主要是请求——应答模式，因此暂时规定client负责request，server负责reply

### request Json
```json 
{
    "version":"v1.0",
    "cmd":"model_cmd",
    "data":{
        "":"",
    };

}


 cmd="user_login",
                data={"username": username, "password": password}
| verison | cmd | data |
|-------|-------|------|
| v1.0  | user

cmd: 模块（user/sample/task/result/report/customer/equipment）
data:
    {
        "username":"....",
    }


```

### reply Json
```json
{
"code": 200, // 状态码，标识请求处理结果，如200表示成功，其他码表示不同的错误或状态
"message": "success", // 提示信息，对状态码进行文字说明
"data":  // 核心数据，根据请求内容返回相应的数据，可为空对象

}

code
编码区间	核心含义	层级	适用场景
2xx	成功类	系统级	通用成功、创建成功、批量操作成功等
3xx	处理中 / 异步类	业务级	异步任务未完成（如批量导入、报告生成）
4xx	客户端错误类	系统级→业务级	参数错误、权限不足、资源不存在、操作非法
5xx	服务端错误类	系统级	数据库异常、第三方接口调用失败、系统故障
6xx~9xx	LIMS 专属业务错误类	业务级→场景级	样本流转异常、检测数据校验失败、审核驳回等

状态码	核心说明	适用场景	关联 HTTP 状态码
200	操作成功（通用）	查询、修改、删除等同步操作成功	200 OK
201	创建成功	样本登记、订单创建、检测任务新增成功	201 Created
202	批量操作部分成功	批量导入样本时部分数据合法、部分失败	200 OK
301	异步任务处理中	报告生成、批量数据导出、仪器数据同步中	202 Accepted
302	异步任务已排队	高并发场景下（如峰值批量提交）任务入队	202 Accepted
400	客户端参数错误（通用）	请求 JSON 格式非法、必填字段缺失、数据类型错误	400 Bad Request
401	未授权（未登录 / 令牌失效）	未携带令牌、JWT 过期、登录状态失效	401 Unauthorized
403	权限不足	普通操作员尝试删除样本、非审核人发起审核	403 Forbidden
404	资源不存在	样本 ID 不存在、检测项目未配置、报告编号无效	404 Not Found
405	操作不允许（业务规则限制）	已审核的报告无法修改、已报废的样本无法流转	405 Method Not Allowed
409	资源冲突	样本编号重复、订单号已存在、检测任务已分配	409 Conflict
412	数据校验失败（业务级）	检测结果超出标准限值、样本有效期已过期	412 Precondition Failed
500	服务端通用错误	代码 Bug、未捕获异常、系统配置错误	500 Internal Server Error
501	功能未实现	预留功能、未上线的业务模块	501 Not Implemented
502	第三方接口调用失败	仪器对接超时、ERP/EMR 系统连接失败	502 Bad Gateway
503	服务暂时不可用	系统维护、数据库备份、高并发限流	503 Service Unavailable
601	样本业务异常	样本未接收却发起检测、样本存储位置占用	-
602	检测任务异常	任务已完成无法重新分配、检测方法未配置	-
603	检测数据异常	原始数据格式非法、平行样偏差超标	-
604	审核流程异常	终审后无法驳回、未完成自审直接提交	-
701	库存业务异常	试剂库存不足、标准品已过期	-
702	设备业务异常	仪器未校准无法使用、设备处于维修状态	-
801	报告业务异常	报告模板不存在、电子签章未配置	-
```
