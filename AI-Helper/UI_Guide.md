# LIMS 客户端界面设计指南

本文档用于讨论和确定 LIMS 客户端的主界面布局和功能模块。

## 1. 主界面布局

建议采用现代桌面应用程序的经典布局：

- **左侧导航栏 (Navigation Bar)**: 用于在不同的主功能模块之间切换。
- **主内容区 (Main Content Area)**: 显示当前选中模块的具体界面。
- **顶部状态栏 (Status Bar)**: (可选) 显示当前登录用户、系统时间或通知。

## 2. 核心功能模块 (建议)

以下是建议的核心功能模块，我们可以基于此进行讨论和调整。

### 2.1. 仪表盘 (Dashboard)

- **目的**: 提供信息概览，让用户快速了解关键信息和待办事项。
- **子模块/功能**:
  - `我的任务`:
    - **功能**: 显示分配给当前用户的任务，并按状态分类展示。
    - **展示方式**:
      - `进行中任务`: 列表或卡片形式，展示当前正在处理的任务，包含任务名称、进度、截止日期等。
      - `已完成任务`: 列表或卡片形式，展示已完成的任务，包含任务名称、完成日期。
      - `未开始任务`: 列表或卡片形式，展示尚未开始的任务，包含任务名称、计划开始日期。
    - **交互**: 点击任务可查看详情，支持按优先级、截止日期排序和筛选。
  - `我的实验`:
    - **功能**: 显示用户负责的实验，并按状态分类展示。
    - **展示方式**:
      - `进行中实验`: 列表或卡片形式，展示当前正在进行的实验，包含实验名称、阶段、负责人、进度。
      - `已完成实验`: 列表或卡片形式，展示已完成的实验，包含实验名称、完成日期、结果概要。
      - `未开始实验`: 列表或卡片形式，展示尚未开始的实验，包含实验名称、计划开始日期、目标。
    - **交互**: 点击实验可查看详情，支持按实验类型、状态、日期排序和筛选。

### 2.2. 实验管理 (Experiment Management)

- **目的**: LIMS 的核心，完整追踪一个实验从创建到完成的全过程。
- **子模块/功能**:
  - `创建实验`:
    - **功能**: 提供表单来定义一个新实验的名称、目标、方法等。
    - **展示方式**: 模态对话框或独立页面，包含表单字段：实验名称、目标、开始日期、结束日期、描述、负责人、状态（草稿、进行中、已完成）、附件上传等。
  - `实验列表`:
    - **功能**: 以列表或表格形式展示所有实验，支持搜索、筛选和排序。
    - **展示方式**: 数据表格，列出实验名称、状态、开始日期、结束日期、负责人等关键信息。提供搜索框、筛选器（按状态、日期范围、负责人）和列排序功能。每行有“查看”、“编辑”、“删除”等操作按钮。
  - `查看/编辑详情`:
    - **功能**: 进入特定实验的详细页面，录入实验数据、上传附件、查看结果。
    - **展示方式**: 独立详情页面，分多个标签页（如基本信息、样本、数据、结果、附件、历史记录）。编辑模式下，字段可修改；查看模式下，字段只读。

### 2.3. 样本管理 (Sample Management)

- **目的**: 管理所有进入实验室的样本信息。
- **子模块/功能**:
  - `样本登记`:
    - **功能**: 记录新样本的来源、类型、数量等信息。
    - **展示方式**: 模态对话框或独立页面，包含表单字段：样本ID、类型、来源、数量、存储位置、采集日期、关联实验、备注等。支持批量导入。
  - `样本列表`:
    - **功能**: 查询和追踪样本当前的状态和位置。
    - **展示方式**: 数据表格，列出样本ID、类型、数量、存储位置、状态、关联实验等。提供搜索、筛选和排序。每行有“查看”、“编辑”操作。

### 2.4. 设备管理 (Equipment Management)

- **目的**: 追踪和管理实验室所有设备的状态、维护历史和校准计划。
- **子模块/功能**:
  - `设备列表`:
    - **功能**: 显示所有设备的清单，包括其型号、状态、负责人等。
    - **展示方式**: 数据表格，列出设备名称、型号、制造商、状态（正常、维护中、故障）、上次维护日期、下次校准日期、负责人。提供搜索、筛选和排序。每行有“查看”、“编辑”操作。
  - `维护记录`:
    - **功能**: 记录每次设备的维护、维修和巡检情况。
    - **展示方式**: 设备的详情页面中的一个标签页或独立列表。表格形式，列出维护日期、维护类型、执行人、描述。提供“添加维护记录”按钮。
  - `校准计划`:
    - **功能**: 管理设备的校准周期和提醒。
    - **展示方式**: 设备的详情页面中的一个标签页或独立列表。表格形式，列出校准日期、校准周期、下次校准日期、校准结果。提供“添加校准计划”按钮。

### 2.5. 系统管理 (System Administration)

- **目的**: (通常仅限管理员访问) 用于配置系统和管理用户。
- **子模块/功能**:
  - `用户管理`:
    - **功能**: 添加新用户、分配角色、重置密码等。
    - **展示方式**: 数据表格，列出用户名、角色、邮箱、状态。提供“添加用户”、“编辑”、“重置密码”、“删除”等操作。
  - `权限管理`:
    - **功能**: 定义不同角色可以访问哪些模块和功能。
    - **展示方式**: 树形结构或表格，列出所有模块/功能，并为每个角色提供勾选框，以配置其权限。

## 3. 权限与访问控制

- **策略**: 采用基于角色的访问控制 (RBAC)。客户端不创建多套界面，而是在同一界面中，根据用户角色动态显示/隐藏功能模块和操作。
- **实现**:
  - 用户角色由服务端在登录成功后通过 Token (如 JWT) 返回。
  - 客户端根据角色信息，在渲染界面时判断每个功能入口（如导航按钮、操作按钮）的可见性。
  - 例如，“系统管理”模块的导航入口只对“管理员”角色的用户可见。
