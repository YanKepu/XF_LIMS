# LIMS 客户端界面设计指南

本文档用于讨论和确定 LIMS 客户端的主界面布局和功能模块。

## 1. 主界面布局

建议采用现代桌面应用程序的经典布局：

- **左侧导航栏 (Navigation Bar)**: 用于在不同的主功能模块之间切换。
- **主内容区 (Main Content Area)**: 显示当前选中模块的具体界面。
- **顶部状态栏 (Status Bar)**: (可选) 显示当前登录用户、系统时间或通知。

## 2. 核心功能模块 (建议)

以下是建议的核心功能模块，我们可以基于此进行讨论和调整。

### 2.1. 仪表盘 (Dashboard)

- **目的**: 提供信息概览，让用户快速了解关键信息和待办事项。
- **子模块/功能**:
  - `我的任务`: 显示分配给当前用户的待处理实验或任务。
  - `系统公告`: 显示管理员发布的通知。
  - `近期活动`: 快速查看最近创建或更新的实验。

### 2.2. 实验管理 (Experiment Management)

- **目的**: LIMS 的核心，完整追踪一个实验从创建到完成的全过程。
- **子模块/功能**:
  - `创建实验`: 提供表单来定义一个新实验的名称、目标、方法等。
  - `实验列表`: 以列表或表格形式展示所有实验，支持搜索、筛选和排序。
  - `查看/编辑详情`: 进入特定实验的详细页面，录入实验数据、上传附件、查看结果。

### 2.3. 样本管理 (Sample Management)

- **目的**: 管理所有进入实验室的样本信息。
- **子模块/功能**:
  - `样本登记`: 记录新样本的来源、类型、数量等信息。
  - `样本列表`: 查询和追踪样本当前的状态和位置。

### 2.4. 设备管理 (Equipment Management)

- **目的**: 追踪和管理实验室所有设备的状态、维护历史和校准计划。
- **子模块/功能**:
  - `设备列表`: 显示所有设备的清单，包括其型号、状态、负责人等。
  - `维护记录`: 记录每次设备的维护、维修和巡检情况。
  - `校准计划`: 管理设备的校准周期和提醒。

### 2.5. 系统管理 (System Administration)

- **目的**: (通常仅限管理员访问) 用于配置系统和管理用户。
- **子模块/功能**:
  - `用户管理`: 添加新用户、分配角色、重置密码等。
  - `权限管理`: 定义不同角色可以访问哪些模块和功能。

## 3. 权限与访问控制

- **策略**: 采用基于角色的访问控制 (RBAC)。客户端不创建多套界面，而是在同一界面中，根据用户角色动态显示/隐藏功能模块和操作。
- **实现**:
  - 用户角色由服务端在登录成功后通过 Token (如 JWT) 返回。
  - 客户端根据角色信息，在渲染界面时判断每个功能入口（如导航按钮、操作按钮）的可见性。
  - 例如，“系统管理”模块的导航入口只对“管理员”角色的用户可见。
